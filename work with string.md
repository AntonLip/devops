
## 1. Теория: базовые операции со строками

### 1.1. Длина строки

Оператор `${#var}` возвращает длину строки в символах:

```bash
msg="user=alice action=login"
echo "Длина: ${#msg}"
```

Это полезно для грубой проверки «пустая/не пустая строка» или для обрезки по длине.

### 1.2. Извлечение подстроки: `${var:pos:len}`

Форма:

- `${var:pos}` — взять подстроку с позиции `pos` до конца;
- `${var:pos:len}` — взять `len` символов, начиная с `pos`.

Индексация с нуля:

```bash
ts="2026-02-01 12:00:00"
date="${ts:0:10}"   # "2026-02-01"
time="${ts:11:8}"   # "12:00:00"
```

Можно использовать отрицательные индексы (от конца строки), но для начала хватает положительных.

### 1.3. Удаление префикса и суффикса: `${var#...}`, `${var%...}`

Эти операции принимают **шаблон glob**, а не регулярное выражение:

- `${var#pattern}` — удалить **минимальный** префикс, подходящий под `pattern`;
- `${var##pattern}` — удалить **максимальный** префикс;
- `${var%pattern}` и `${var%%pattern}` аналогично, но с конца строки.

Примеры:

```bash
path="/var/log/sample.log"
echo "${path#/var/log/}"   # sample.log

entry="user=alice action=login"
echo "${entry#user=}"      # alice action=login (срезали префикс до первого пробела)
echo "${entry% action=*}"  # user=alice         (срезали «хвост» начиная с " action=")
```

### 1.4. Разбиение строки: `IFS` и `read`

Частая задача — взять строку и разнести её по нескольким переменным. Для этого используют `read`:

```bash
line="2026-02-01 12:00:00 [INFO] user=alice action=login"

IFS=' ' read -r date time level rest <<< "$line"

echo "Дата: $date"
echo "Время: $time"
echo "Уровень: $level"
echo "Остальное: $rest"
```

Здесь:

- `IFS=' '` — разделяем по пробелам;
- `date` — первое поле;
- `time` — второе поле;
- `level` — третье поле;
- `rest` — всё оставшееся.

Для разбора всего файла удобно использовать цикл:

```bash
while IFS= read -r line; do
    # обработка $line
done < sample.log
```

### 1.5. Проверка наличия подстроки

Самый простой способ проверить, содержит ли строка подстроку — использовать шаблон сравнения:

```bash
msg="user=alice action=login"

if [[ "$msg" == *"alice"* ]]; then
    echo "Нашли alice"
fi
```

Шаблоны в `[[ ... == ... ]]` — это glob (звёздочка, вопрос и т.д.), этого достаточно, если не нужны сложные регулярные выражения.

### 1.6. awk и sed — разбор и замена текста

Кроме встроенных возможностей Bash для работы со строками часто используют классические утилиты, например: **awk** и **sed**. Они обрабатывают поток строк (файл или stdin) и удобны для логов и конфигов.

#### awk

**awk** разбивает каждую строку на поля (по умолчанию — по пробелам/табуляции) и позволяет обращаться к ним как к `$1`, `$2`, `$3` и т.д. `$0` — вся строка.

- Задать разделитель полей: `-F'разделитель'` (например `-F:` для двоеточия, `-F'[][]'` для `[` или `]`).
- Формат: `условие { действие }`. Если условие не указано — действие выполняется для каждой строки.
- Встроенные переменные: `NF` — число полей в текущей строке, `NR` — номер строки.

Примеры для формата лога:

```bash
# Вывести только дату (поле 1) и уровень (поле 3)
awk '{print $1, $3}' sample.log

# Разбить по пробелу и квадратным скобкам: тогда уровень — в отдельном поле
awk -F'[][]' '{print $1, $2}' sample.log
# $1 = "2026-02-01 12:00:00 ", $2 = "INFO" или "ERROR" и т.д.

# Вывести строки, где третье поле содержит ERROR
awk '$3 ~ /ERROR/' sample.log
```

Для сложного разбора (несколько разделителей, арифметика, счётчики) awk удобнее, чем только Bash.

#### sed

**sed** (stream editor) предназначен в первую очередь для **замены** текста по шаблону и удаления/вставки строк.

- Замена: `sed 's/что/на_что/'` — заменить первое вхождение в каждой строке; флаг `g` в конце — все вхождения: `s/старое/новое/g`.
- Разделитель можно менять, если в паттерне много слэшей: `sed 's|/var/log|/opt/log|'`.
- Удаление строк: `sed '/шаблон/d'` — удалить строки, подходящие под шаблон.
- Номера строк: `sed '2,5d'` — удалить строки 2–5; `sed '3s/old/new/'` — замена только в третьей строке.

Примеры:

```bash
# Убрать квадратные скобки вокруг уровня: [INFO] -> INFO
echo "2026-02-01 12:00:00 [INFO] user=alice" | sed 's/\[\(.*\)\]/\1/'

# Заменить в файле все ERROR на CRITICAL
sed 's/ERROR/CRITICAL/g' sample.log

# Вывести только первые 10 строк (аналог head -10)
sed -n '1,10p' sample.log
# -n отключает печать по умолчанию, 1,10p — печатать строки 1–10
```

В скриптах часто комбинируют: **Bash** — циклы, условия, вызовы; **awk** — разбор полей и подсчёты; **sed** — быстрые замены и фильтрация по номеру строки.

## 2. Практика: дорабатываем `log_analyzer.sh` (разбор строк)

В этой части мы научим скрипт:

- для каждой строки лога выделять дату, время, уровень и «сообщение»;
- выводить более читаемое резюме через функцию `print_summary`;
- использовать функции для разбора строки вместо «сырых» `grep`.

### 2.1. Функция `parse_log_line`

Добавляем функцию разбора одной строки лога. Полный скрипт `log_analyzer.sh` на этом шаге:

```bash
#!/usr/bin/env bash

usage() {
    cat <<EOF
Использование: $0 LOG_FILE [LEVEL]

LOG_FILE — путь к файлу лога.
LEVEL    — необязательный уровень: INFO, WARN, ERROR.

Примеры:
  $0 sample.log
  $0 sample.log ERROR
EOF
}

check_file_exists() {
    local path="$1"
    if [ ! -f "$path" ]; then
        echo "Ошибка: файл не найден: $path" >&2
        return 1
    fi
}

parse_log_line() {
    local line="$1"

    # Разносим строку на 4 части: дата, время, уровень, остальное
    local date time level rest
    IFS=' ' read -r date time level rest <<< "$line"

    # Если строка пустая или не похоже на лог — просто выходим
    if [ -z "$date" ] || [ -z "$time" ] || [ -z "$level" ]; then
        return 0
    fi

    # Удалим квадратные скобки вокруг уровня: [INFO] -> INFO
    level="${level#[}"
    level="${level%]}"

    # Остальное считаем «сообщением»
    local message="$rest"

    echo "Дата: $date, Время: $time, Уровень: $level, Сообщение: $message"
}

filter_by_level() {
    local level="$1"
    local file="$2"

    case "$level" in
        ALL)
            cat "$file"
            ;;
        INFO|WARN|ERROR)
            grep "[$level]" "$file"
            ;;
        *)
            echo "Неизвестный уровень: $level" >&2
            return 1
            ;;
    esac
}

main() {
    local log_file="$1"
    local level="${2:-ALL}"

    if [ -z "$log_file" ]; then
        echo "Ошибка: не указан LOG_FILE" >&2
        usage
        exit 1
    fi

    check_file_exists "$log_file" || exit 1

    echo "Лог-файл: $log_file"
    echo "Уровень:  $level"
    echo "---- Начало вывода ----"
    filter_by_level "$level" "$log_file"
    echo "---- Конец вывода ----"
}

main "$@"
```

Пояснения:

- `read -r date time level rest <<< "$line"` — подаём строку на вход `read`, получаем четыре поля по пробелам;
- `${level#[}` и `${level%]}` — удаляют первую `[` и последнюю `]` в уровне;
- `parse_log_line` пока только добавлена, в выводе её задействуем в следующих шагах.

### 2.2. Функция `print_summary` для одной строки

Добавляем функцию, которая вызывает `parse_log_line` и печатает результат. Полный скрипт:

```bash
#!/usr/bin/env bash

usage() {
    cat <<EOF
Использование: $0 LOG_FILE [LEVEL]

LOG_FILE — путь к файлу лога.
LEVEL    — необязательный уровень: INFO, WARN, ERROR.

Примеры:
  $0 sample.log
  $0 sample.log ERROR

}

    local line="$1"

    local date time level rest
    IFS=' ' read -r date time level rest <<< "$line"

    if [ -z "$date" ] || [ -z "$time" ] || [ -z "$level" ]; then

    level="${level%]}"

}

print_summary() {

    parsed="$(parse_log_line "$line")"

    [ -z "$parsed" ] && return 0

    echo "$parsed"
}

filter_by_level() {
    local level="$1"
    local file="$2"

    case "$level" in
        ALL)
            cat "$file"
            ;;
        INFO|WARN|ERROR)

    echo "---- Конец вывода ----"
}

main "$@"
```

Два слоя: `parse_log_line` разбирает строку, `print_summary` форматирует и печатает. В следующем шаге `filter_by_level` будет вызывать `print_summary` вместо вывода «сырых» строк.

### 2.3. Используем разбор в `filter_by_level`

Меняем `filter_by_level`: вместо `grep`/`cat` читаем файл построчно, фильтруем по уровню и для каждой подходящей строки вызываем `print_summary`. Полный скрипт:

```bash
#!/usr/bin/env bash

usage() {
    cat <<EOF
Использование: $0 LOG_FILE [LEVEL]

LOG_FILE — путь к файлу лога.
LEVEL    — необязательный уровень: INFO, WARN, ERROR.

Примеры:
  $0 sample.log
  $0 sample.log ERROR
EOF
}

check_file_exists() {
    local path="$1"
    if [ ! -f "$path" ]; then
        echo "Ошибка: файл не найден: $path" >&2
        return 1
    fi
}

parse_log_line() {
    local line="$1"

    local date time level rest
    IFS=' ' read -r date time level rest <<< "$line"

    if [ -z "$date" ] || [ -z "$time" ] || [ -z "$level" ]; then
        return 0
    fi

    level="${level#[}"
    level="${level%]}"
    local message="$rest"

    echo "Дата: $date, Время: $time, Уровень: $level, Сообщение: $message"
}

print_summary() {
    local line="$1"
    local parsed

    parsed="$(parse_log_line "$line")"

    [ -z "$parsed" ] && return 0

    echo "$parsed"
}

filter_by_level() {
    local level="$1"
    local file="$2"

    if [ "$level" = "ALL" ]; then
        while IFS= read -r line; do
            print_summary "$line"
        done < "$file"
        return 0
    fi

    case "$level" in
        INFO|WARN|ERROR)
            while IFS= read -r line; do
                if [[ "$line" == *"[$level]"* ]]; then
                    print_summary "$line"
                fi
            done < "$file"
            ;;
        *)
            echo "Неизвестный уровень: $level" >&2
            return 1
            ;;
    esac
}

main() {
    local log_file="$1"
    local level="${2:-ALL}"

    if [ -z "$log_file" ]; then
        echo "Ошибка: не указан LOG_FILE" >&2
        usage
        exit 1
    fi

    check_file_exists "$log_file" || exit 1

    echo "Лог-файл: $log_file"
    echo "Уровень:  $level"
    echo "---- Начало вывода ----"
    filter_by_level "$level" "$log_file"
    echo "---- Конец вывода ----"
}

main "$@"
```

Фильтрация по уровню сделана через шаблон `[[ "$line" == *"[$level]"* ]]` (glob).

## Справочник: команды и параметры

Краткая расшифровка команд, часто встречающихся в решениях выше.


| Команда  | Основные параметры     | Назначение                                                                                    |
| -------- | ---------------------- | --------------------------------------------------------------------------------------------- |
| **awk**  | `'{print $N}'`         | Вывести N-е поле строки (разделитель по умолчанию — пробел/табуляция).                        |
|          | `-F'X'` или `-F'[][]'` | Задать разделитель полей (символ или несколько).                                              |
|          | `$9 ~ /regex/`         | Условие: поле $9 совпадает с регулярным выражением.                                           |
|          | `split(s,a," ")`       | Разбить строку `s` по разделителю в массив `a`.                                               |
|          | `END{...}`             | Блок выполняется после обработки всех строк.                                                  |
| **grep** | `-E`                   | Расширенные регулярные выражения (ERE).                                                       |
|          | `-i`                   | Поиск без учёта регистра.                                                                     |
|          | `-c`                   | Выводить только количество совпадающих строк.                                                 |
|          | `-o`                   | Выводить только совпавшую часть строки.                                                       |
| **sort** | (без опций)            | Сортировка по алфавиту (нужен перед `uniq`).                                                  |
|          | `-n`                   | Сортировка как чисел.                                                                         |
|          | `-r`                   | В обратном порядке (по убыванию).                                                             |
|          | `-k N`                 | Сортировать по N-му полю.                                                                     |
| **uniq** | `-c`                   | Перед каждой уникальной строкой вывести число её повторений (требуется отсортированный ввод). |
| **head** | `-n N` или `-N`        | Вывести только первые N строк.                                                                |
| **tail** | `-n N`                 | Вывести только последние N строк.                                                             |
| **cut**  | `-d:X`                 | Разделитель полей — X (например `-d:`).                                                       |
|          | `-f N` или `-f N-M`    | Номера полей (одно или диапазон).                                                             |
| **wc**   | `-l`                   | Количество строк; `wc -l < file` — без имени файла в выводе.                                  |
| **sed**  | `'s/что/на что/"`      | Заменить первое вхождение подстроки в каждой строке.                                          |
| **tr**   | `-d 'набор'`           | Удалить из ввода все символы из набора.                                                       |
| **Bash** | `${var:-значение}`     | Подстановка: если переменная пуста или не задана — использовать значение.                     |
|          | `${line#*X}`           | Удалить минимальный префикс до первого X.                                                     |
|          | `${line##*X}`          | Удалить максимальный префикс до последнего X.                                                 |
|          | `${line%X*}`           | Удалить минимальный суффикс после последнего X.                                               |
|          | `${line%%X*}`          | Удалить максимальный суффикс после первого X.                                                 |
|          | `$(( a + b ))`         | Арифметика в shell.                                                                           |
|          | `[ ! -f "$f" ]`        | Условие: файл не существует; `-f` — обычный файл.                                             |
